# TabICL v2 实验报告

## 1. 实验目的

本实验旨在验证表格基础模型 TabICL（Tabular In-Context Learning）在结构化数据分类任务中的表现，并从以下三个方面进行评估：

1. 基础分类性能
2. 缺失值鲁棒性
3. 数据规模可扩展性

实验结果将与传统机器学习基线模型（XGBoost、LightGBM）进行后续对比分析。

---

| 项目 | 配置 |
|--------|--------|
| 操作系统 | Windows 10 |

| 深度学习框架 | PyTorch |
| 模型 | TabICL v2 |
| 随机种子 | 42 |
| 运行设备 | CPU |

## 3. 数据集

实验采用三个公开表格数据集：

| 数据集 | 类型 |
|---------|---------|
| Adult | 收入预测 |
| Credit-G | 德国信用评估 |
| Covtype | 森林覆盖类型分类 |

由于 Covtype 数据集规模较大，在本机 CPU 环境下运行 TabICL 时出现内存不足问题，因此仅完成 Adult 与 Credit-G 的实验。

---

### Credit-G 数据集内存占用说明

在运行 TabICL 时发现，Credit-G 数据集即使样本数（700）少于 Adult（7000），峰值内存却显著更高（约 7.5 GB vs 0.9 GB）。原因分析如下：

1. **特征维度更高**  
   Credit-G 数据集包含更多特征（尤其是类别型特征），每个类别在模型中会被编码为 embedding，导致内部矩阵占用较大内存。

2. **TabICL 模型计算特性**

TabICL 使用 embedding 与注意力机制对表格数据进行建模，其内存开销不仅与样本数量有关，也受到特征维度、类别特征数量以及内部表示维度等因素影响。

因此，即使 Credit-G 的样本数少于 Adult，由于数据特征结构不同，仍可能产生更高的内存占用。

3. **结果说明**  
   - Adult: 训练集 7000 × 少量特征 → 内存约 0.9 GB  
   - Credit-G: 训练集 700 × 特征多 → 内存约 7.5 GB  

4. **结论**  
   内存消耗差异是模型特性造成的，而非代码错误。若需要处理更大规模数据集，可考虑：
   - 使用 GPU 或更大内存机器
   - 对数据进行降维或裁剪特征
   - 对 TabICL 实现分批训练

## 5. 缺失值鲁棒性实验

### 5.1 Adult

| 缺失率 | Accuracy | F1 | AUC |
|----------|----------|----------|----------|
| 0% | 0.8623 | 0.7971 | 0.9197 |
| 10% | 0.8573 | 0.7889 | 0.9111 |
| 20% | 0.8407 | 0.7635 | 0.9022 |
| 30% | 0.8460 | 0.7690 | 0.8964 |

### 5.2 Credit-G

| 缺失率 | Accuracy | F1 | AUC |
|----------|----------|----------|----------|
| 0% | 0.7700 | 0.6891 | 0.7879 |
| 10% | 0.7567 | 0.6770 | 0.7897 |
| 20% | 0.7333 | 0.6444 | 0.7702 |
| 30% | 0.7133 | 0.5868 | 0.7060 |

### 5.3 结果分析

随着缺失率增加，TabICL整体性能呈下降趋势。

- Adult数据集Accuracy从0.8623下降至0.8460；
- Credit-G数据集Accuracy从0.7700下降至0.7133；
- F1指标下降趋势更加明显。

值得注意的是，在Adult数据集上，30%缺失率下的Accuracy（0.8460）略高于20%缺失率（0.8407）。由于两者差异较小（约0.5个百分点），更可能来源于随机缺失值生成带来的统计波动，而非模型性能的真实提升。因此总体上仍可认为，随着缺失比例增加，模型性能呈下降趋势。

实验结果表明，TabICL对缺失值具有一定鲁棒性，但在高缺失率场景下性能仍会受到明显影响。

- 在 CPU 环境下资源开销较大。

## 7. Covtype 实验说明

Covtype 数据集预处理后训练集规模约为 35000 条样本。

运行 TabICL 时出现如下错误：

```text
RuntimeError:
DefaultCPUAllocator: not enough memory
you tried to allocate 11673600000 bytes
```

模型尝试申请约 11.7GB 连续内存，超过实验设备可用内存容量。

因此：

- Basic 实验未完成；
- Missing 实验未完成；
- Scalability 实验未完成；

本项目最终仅保留 Adult 与 Credit-G 的实验结果，并在报告中说明硬件限制原因。

---

## 8. 实验结论

本实验验证了 TabICL 在结构化数据分类任务中的性能表现。

主要结论如下：

1. TabICL 在 Adult 数据集上取得较高分类性能，AUC 超过 0.91。
2. 缺失值比例增加时模型性能逐步下降，但整体保持一定鲁棒性。
3. 增加训练样本数量能够提升模型性能。
4. TabICL 的内存占用随样本规模快速增长。
5. 在本实验设备（CPU 环境）下，TabICL 无法完成 Covtype 数据集训练，说明其资源消耗较高，对硬件配置要求较为敏感。
6. 后续可与 TabPFN、XGBoost 和 LightGBM 的实验结果进一步比较，以分析表格基础模型与传统机器学习模型之间的差异。
